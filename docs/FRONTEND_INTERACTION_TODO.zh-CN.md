# EasySession 前端交互与操作体验 TODO

> 生成日期：2026-06-07  
> 适用范围：`D:\EasySession`  
> 编码要求：UTF-8  
> 目标：从用户操作、前端体验、交互安全、信息架构和桌面效率出发，系统性收敛 EasySession 当前的可用性问题。

## 0. 总原则

- [ ] 优先保护用户数据与会话资产，任何删除、丢弃、同步、重启类操作都必须可理解、可确认、可恢复或可撤销。
- [ ] 优先服务主工作流：进入项目、创建/恢复会话、查看终端、切换分窗、查看变更。
- [ ] 默认界面保持克制，高级能力通过渐进披露出现。
- [ ] 所有图标按钮必须有 tooltip、aria-label、禁用原因或清晰状态。
- [ ] 不再新增只有开发者能猜懂的缩写按钮。
- [ ] 不依赖旧截图做判断，以当前代码与真实运行体验为准。
- [ ] 所有交互改动必须保持本地、远程挂载、远程 Web 的边界不串。
